# Aop思想

Aop是面向切面编程，它的作用就是实现功能增强。

我们重点来看一下为什么需要Aop？

在开发时，一些需求，比如说：系统日志、性能统计等。分布在各个接口，编写和维护起来很麻烦。

而且这种问题也不是oop能够解决的。

于是Aop横空出世。

这样做有两点好处：

1. 简化了开发和维护。
2. 把非核心功能抽取出来形成切面，可在不修改业务代码时对这些功能进行修改或者增强。
